# weiji-web · 功能参考原型（Prototype）

> ⚠️ **本目录是「功能参考原型设计」，不是正式工程。**
>
> 它**不参与构建、部署、测试**，也**不被任何端（weiji-server / weiji-admin-web / weiji-app）依赖**。
> 它的唯一作用：作为味记产品功能、交互、视觉、数据模型的**事实标准参考**，供三端基于 cool-admin 全家桶重写时对照。

---

## 定位

| 维度 | 说明 |
|------|------|
| **是什么** | 纯静态 HTML + 原生 JS 的可交互原型（单页应用，`index.html` + `app.js` + `api.js`，无后端依赖，`api.js` 模拟接口） |
| **为什么保留** | 功能覆盖最全（30+ 业务、7 个页面、完整数据模型），是 PRD 的可运行化身 |
| **正式实现** | 见架构设计：[../架构设计与迁移方案.md](../架构设计与迁移方案.md) |

---

## 参考价值清单（给三端重写者）

### 页面结构（7 页）
首页、记录页、家庭菜谱、成就徽章、趣味玩法、个人中心、登录

### 业务功能（对应 PRD）
- **记录域**：拍照记录、AI 食物识别、营养分析、图片美化、评分/标签/餐次、日记列表与日历视图
- **家庭域**：菜谱 CRUD + 可见性、本周菜单 + 投票、购物清单协同、家庭动态 + 点赞评论、家庭报告
- **成就域**：等级/经验、徽章解锁、连续打卡 + 补签、周期挑战赛
- **玩法域**：美食图鉴、食物人格测试、美食时光机、家庭盲猜游戏
- **其他**：语音交互、个性化推荐、用户资料

### 数据模型样板
record / recipe / family / member / weeklyMenu / shoppingItem / achievement / level / challenge 等 9 类实体的字段结构，可直接作为后端 entity 与数据库表设计的参照。详见各页面的 mock 数据与 `api.js`。

### 交互与视觉
- 移动优先响应式（移动端底部 Tab + 桌面端侧栏）
- 主色橙红（`#ff5e1a`）+ 辅助金黄（`#f5a623`）
- 组件范式：Toast、模态对话框、进度条、卡片、标签、头像状态

---

## 如何使用

1. **重写某模块时**：先打开对应页面，对照功能点、字段、交互流，再在 cool-admin 工程中实现
2. **本地预览**：直接浏览器打开 `index.html` 即可（无需构建）
3. **不要**：把这里的代码拷进正式工程；这里用的是原生 JS，正式工程用 Vue3/uni-app
