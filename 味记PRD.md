## 1. 需求背景与市场机会

### 1.1 业务背景

中国菜谱App行业2025年市场规模达 **48.6亿元**，同比增长12.3%，付费用户突破1,286万人。家庭厨房数字化渗透率从2024年的29.1%跃升至2025年的38.7%，下沉市场渗透率从28.4%升至35.7%。**家庭场景正在成为美食App的核心使用场景**，而现有产品均围绕"公开菜谱社区"定位，未覆盖家庭饮食记录与协作这一显著空白。

| 指标 | 数值 |
|------|------|
| 2025年市场规模 | 48.6亿元 |
| 同比增长率 | 12.3% |
| 家庭厨房数字化渗透率 | 38.7% |
| 付费用户规模 | 1,286万 |
| 60岁以上用户占比 | 18.3% |

### 1.2 用户痛点

现有产品存在五大核心痛点，构成新产品的差异化切入点：

| 痛点 | 说明 | 证据强度 |
|------|------|----------|
| 收藏不等于记录 | 87%用户看完做菜视频不会立刻动手，收藏夹越满越焦虑；小红书/抖音收藏的内容无法结构化，原内容下架即永久丢失 | 数据支撑 |
| 配方无法迭代 | 无法记录"这次盐少放了半勺"等个人调整，下厨房3,270万道菜谱均为静态公开版本 | 调研支撑 |
| 家庭协作缺失 | 主流App几乎无多人共享菜谱空间、协作点餐能力；家庭烹饪天然涉及多人协作和长期沉淀 | 调研支撑 |
| 跨平台断层 | 小红书/抖音收藏无法结构化，跨平台内容无法整合为个人饮食知识库 | 调研支撑 |
| 记录工具薄弱 | 现有产品以"内容消费"为核心，缺乏以"个人记录"为第一优先级的工具设计 | PM判断 |

### 1.3 不解决的后果

如果继续由现有产品覆盖用户需求：用户的饮食记忆持续碎片化，家庭饮食文化无法数字化沉淀，银发族（60岁以上用户占比已达18.3%）持续被主流产品忽视。而随着AI技术成熟（多模态识别准确率达98.7%），"拍照即记录"的零门槛体验已具备技术可行性，窗口期有限。

## 2. 竞品分析

### 2.1 竞品定位对比

| 竞品 | 产品定位 | 目标用户 | MAU | 核心能力 |
|------|----------|----------|-----|----------|
| 下厨房 | UGC菜谱社区+电商 | 烹饪爱好者 | 2,180-2,470万 | 3,270万道菜谱，AI营养管家 |
| 豆果美食 | 美食社交+VIP体系 | 家庭烹饪者 | 1,650万 | 社区交流，多品类电商 |
| 大众点评 | 本地生活服务平台 | 外出就餐用户 | 1.86亿 | 餐厅评价，到店引流 |
| 小红书 | 生活方式内容社区 | 年轻女性为主 | 食品人群1.5亿+ | 种草内容，视频化表达 |
| Noms | AI贴纸美食日记 | Z世代个人用户 | 新兴产品 | AI贴纸生成，物理互动 |

### 2.2 核心功能对比

| 维度 | 下厨房 | 豆果美食 | 大众点评 | 小红书 | Noms | **味记** |
|------|--------|----------|----------|--------|------|----------|
| 核心问题 | 这道菜怎么做？ | 今天做什么？ | 这家店好不好？ | 怎么拍好看？ | 今天吃了什么？ | **我家吃过什么？** |
| 内容中心 | 菜谱 | 菜谱+社区 | 餐厅 | 生活方式 | 个人日记 | **人 & 家庭** |
| AI能力 | AI营养管家 | 智能推荐 | 口味图谱 | 推荐算法 | AI贴纸 | **全链路AI** |
| 家庭场景 | 弱 | 弱 | 无 | 无 | 无 | **核心场景** |
| 记录方式 | 作品发布 | 写评价 | 写评价 | 发笔记 | 拍照记录 | **拍照即记录** |
| 社交单元 | 作品展示 | 社区交流 | 评价晒单 | 种草笔记 | 密友分享 | **家庭协作** |

### 2.3 竞品分析结论

**核心发现：** 现有头部产品均聚焦"公开菜谱社区"或"餐厅评价"，而 **"私人/家庭饮食记录中心"这一细分场景几乎空白**。下厨房回答"这道菜怎么做"，大众点评回答"这家店好不好吃"，没有任何产品真正回答"我家常吃什么？谁爱吃什么？今天从哪些菜里选？"——这是整个市场最显著的空白点。

味记的差异化策略：**不跟下厨房拼菜谱数量，不跟大众点评拼餐厅覆盖，不跟小红书拼内容社区**——聚焦"以人和家庭为中心的个人饮食记录"，通过AI降低记录门槛，通过家庭组构建场景壁垒。

## 3. 产品定位与用户画像

### 3.1 产品定位

> **一句话定位：** 味记是一个以个人和家庭为主体的AI美食记录平台——记录每一天吃到的美食，无论来自自家厨房、街边小摊还是米其林餐厅。  
> **核心理念：** 每一餐都值得被记住，每一个家庭都有自己的味道。

### 3.2 做什么与不做什么

| 做什么 | 不做什么 | 原因 |
|--------|----------|------|
| 个人/家庭美食日记与记录 | 公开菜谱社区 | 下厨房有3,270万道菜谱，重复没有意义 |
| AI拍照识图+美化+推荐+营养分析 | 餐厅点评平台 | 大众点评月活1.86亿，正面竞争无胜算 |
| 家庭共享菜谱+协作菜单+饮食报告 | 美食内容社区 | 小红书食品人群1.5亿+，内容竞争已白热化 |
| 轻度游戏化娱乐（成就/挑战/贴纸） | 外卖/团购平台 | 美团饿了么把持市场，壁垒极高 |
| 银发关怀模式（大字版+语音+适老食谱） | 复杂社交网络 | 不做朋友圈式信息流，专注家庭小圈层 |

### 3.3 目标用户画像

| 画像 | 描述 | 使用频率 | 核心诉求 | 痛点严重度 |
|------|------|----------|----------|------------|
| **小王：都市美食爱好者** | 25-35岁上班族，热爱探索美食，日常拍照记录午餐和晚餐，偶尔自己下厨 | 每日2-3次 | 快速记录、AI自动分析、美食日记回顾 | 中 |
| **李妈妈：家庭主厨** | 30-45岁，负责全家饮食，需要规划每周菜单、管理家庭菜谱、同步购物清单 | 每日多次 | 家庭协作、菜单规划、菜谱传承 | 高 |
| **张奶奶：银发用户** | 60岁以上，有轻度慢性病，需要大字版、语音交互、低盐低糖食谱推荐 | 每日1-2次 | 语音交互、健康饮食、家人关注 | 高 |
| **小陈：大学生美食探索者** | 18-24岁，喜欢探索校园周边美食，参与挑战赛，分享美食地图 | 每日1-2次 | 娱乐化体验、社交分享、美食探索 | 低 |

### 4.3 核心模块详细规格

#### 4.3.1 拍照记录（F1 + F2）

**业务流程：**

```
拍照/选照片 → 上传AI识别（1-3秒Loading） → 识别结果展示 → 用户确认/编辑 → 保存记录
```

**交互逻辑：**

| 步骤 | 操作 | 系统响应 |
|------|------|----------|
| 1 | 点击拍照按钮 | 调起系统相机或相册选择器 |
| 2 | 确认照片 | 上传至AI识别服务，显示Loading动画 |
| 3 | 识别完成 | 展示识别结果页：菜品名称、食材清单、烹饪方式、AI美化预览 |
| 4 | 用户编辑 | 可编辑任意字段，选择评分和标签 |
| 5 | 点击保存 | Toast提示"记录成功"，跳转至当日日记页 |
| 6 | 点击放弃 | 弹出确认弹窗："放弃本次记录？" |

**规则约束：**

| 字段 | 约束 |
|------|------|
| 菜品名称 | 文本，1-50字，必填，不允许纯数字或纯符号 |
| 评分 | 1-5星整数，默认3星 |
| 标签 | 最多5个，预设标签+自定义标签（自定义标签≤10字） |
| 照片 | 支持JPG/PNG/HEIC，≤20MB，自动压缩至≤2MB存储 |
| 备注 | 可选，≤500字 |

**边界与异常：**

| 场景 | 处理 |
|------|------|
| 网络超时（>5秒） | 显示"网络不稳定，请重试"，提供重试按钮 |
| 识别服务不可用 | 降级为手动输入模式，提示"AI识别暂不可用，请手动记录" |
| 照片非食物 | 识别结果置信度<50%时，提示"这可能不是食物照片，是否继续记录？" |
| 并发上传 | 同一用户同时只能处理一条记录，新请求排队等待 |

#### 4.3.2 家庭组 — 共享菜谱空间（F13）

**业务流程：**

```
创建家庭组 → 邀请成员（最多6人） → 上传菜谱 → 分类/标签 → 成员浏览/评论 → 编辑/迭代
```

**权限逻辑：**

| 角色 | 权限 |
|------|------|
| 管理员（创建者） | 邀请/移除成员、编辑/删除任意菜谱、修改家庭组名称 |
| 普通成员 | 上传菜谱、编辑/删除自己的菜谱、评论他人菜谱 |
| 被移除成员 | 其上传的菜谱保留，标记为"已离开成员" |

**规则约束：**

| 字段 | 约束 |
|------|------|
| 家庭组规模 | 2-6人 |
| 菜谱名称 | 2-50字，必填 |
| 食材清单 | 至少1项，每项含名称+用量+单位 |
| 步骤 | 至少1步，每步≤500字+可选配图 |
| 分类 | 至少选1个，支持自定义分类 |

**边界与异常：**

| 场景 | 处理 |
|------|------|
| 家庭组满员 | 邀请按钮置灰，提示"家庭组已满" |
| 重复菜谱 | 上传时检测同名菜谱，提示"已存在同名菜谱，是否合并？" |
| 网络异常 | 编辑内容自动暂存本地，恢复网络后提示同步 |

#### 4.3.3 AI贴纸日记与成就系统（F12 + F21 + F22）

**成就系统设计：**

| 机制 | 设计 |

## 5. AI功能规格

### 5.1 AI任务定义

| 任务 | 输入 | 输出 | 延迟目标 | 并发目标 |
|------|------|------|----------|----------|
| 菜品图像识别 | 美食照片（JPG/PNG/HEIC，≤20MB，压缩后≤2MB） | 菜品名称、食材清单（含置信度）、烹饪方式、置信度 | P95 ≤ 3秒 | QPS 500 |
| AI图片美化 | 美食照片 + 风格选择 | 美化后图片（保持原分辨率，≤2MB） | P95 ≤ 2秒 | QPS 300 |
| 菜谱推荐 | 菜品名称/食材清单/用户偏好/健康档案 | Top 10推荐菜谱（含菜名、食材、步骤、难度、时间、匹配度） | P95 ≤ 2秒 | QPS 200 |
| 营养分析 | 菜品名称 + 食材清单 + 份量估算 | 热量、蛋白质、脂肪、碳水、膳食纤维、钠等；进阶：65种全营养素 | P95 ≤ 1.5秒 | QPS 200 |
| AI贴纸生成 | 美食照片 + 贴纸风格 | 512×512 PNG贴纸，透明背景 | P95 ≤ 4秒（异步） | QPS 100 |
| 语音交互 | 用户语音（中文，≤30秒） | 转写文本 + 意图识别 + 语音合成回复 | P95 ≤ 3秒（端到端） | QPS 100 |

### 5.2 准确率目标

| AI任务 | 指标 | MVP目标 | 正式版目标 |
|--------|------|---------|------------|
| 菜品识别 | Top-1菜名准确率 | ≥90% | ≥95% |
| 菜品识别 | 食材识别F1 | ≥85% | ≥92% |
| 图片美化 | 用户满意度（5分制） | ≥3.8 | ≥4.2 |
| 菜谱推荐 | Top-5点击率 | ≥30% | ≥45% |
| 营养分析 | 热量误差率 | ≤20% | ≤15% |
| 语音识别 | WER（词错误率） | ≤10% | ≤5% |

### 5.3 评估体系

采用"离线评估 + 人工审核 + 线上反馈"三重评估体系：

| 评估层 | 方法 | 频率 | 责任人 |
|--------|------|------|--------|
| 离线评估 | 标准样本集（5,000+标注图片）、边界样本、负样本、回归样本 | 每次模型更新 | 算法团队 |
| 人工审核 | 每周随机抽取200条AI识别结果，3人独立评分（1-5分），Kappa系数监控一致性 | 每周 | 运营团队 |
| 线上反馈 | 用户编辑率、满意度评分、弃用率、分享率 | 实时监控 | 产品+数据 |

### 5.4 Go/No-Go 标准

| 维度 | Go 条件 | No-Go 红线 |
|------|---------|------------|
| 准确度 | 菜品识别Top-1准确率≥90%，食材识别F1≥85% | 菜品识别准确率<80%，或食材识别F1<70% |
| 体验 | 端到端识别延迟P95≤3秒，用户满意度≥3.8 | P95延迟>5秒，或用户满意度<3.0 |
| 安全 | 不良内容识别准确率≥99%，无合规风险 | 出现任何合规风险事件 |
| 成本 | 单次AI识别成本≤0.05元 | 单次成本>0.15元，或日成本超过预算150% |
| 运维 | AI服务可用性≥99.5%，降级方案可用 | 连续2天可用性<99% |

### 5.5 降级与兜底机制

| 场景 | 降级策略 | 用户感知 |
|------|----------|----------|
| AI识别服务不可用 | 自动切换为手动输入模式 | 提示"AI识别暂不可用，请手动输入菜名" |
| 识别置信度低（<50%） | 展示结果但高亮提示用户确认 | 结果可编辑，用户确认后保存 |
| AI美化服务不可用 | 使用原图，不阻塞记录流程 | 照片正常保存，后续可重新美化 |
| 菜谱推荐超时 | 展示本地缓存通用推荐列表 | 标注"离线推荐"，联网后自动刷新 |
| AI贴纸生成失败 | 使用默认风格兜底，后台异步重试 | 通知"贴纸正在制作中" |
| 语音识别失败 | 提示用户手动输入 | 弹窗"未能识别您的语音，请尝试文字输入" |

### 9.2 关键架构决策

| 数据安全 | 用户数据加密存储，家庭组内数据隔离，支持数据导出 | 饮食记录为个人隐私数据，需遵循个人信息保护法 |

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| AI识别准确率不达标 | 中 | 高 | MVP阶段允许手动编辑兜底，设置置信度阈值提示用户确认；每2周评估一次准确率提升 |
| 用户冷启动困难 | 高 | 高 | 依托微信小程序降低安装门槛；通过家庭邀请机制实现社交裂变；与美食KOL合作进行内容冷启动 |
| 功能过载导致产品臃肿 | 中 | 中 | 严格遵循MVP优先原则，P1/P2功能按数据验证结果分批上线；每季度进行功能使用率评估 |
| AI服务成本失控 | 中 | 中 | 设置单次AI调用成本上限（0.05元/次）；核心模型自建降低边际成本 |
| 巨头模仿进入 | 中 | 中 | 通过家庭组数据沉淀构建迁移壁垒（家庭菜谱数据不可迁移）；聚焦差异化场景 |
| 隐私合规风险 | 低 | 高 | 饮食数据本地化存储+用户授权；隐私政策明确数据使用范围；设立数据删除机制 |
